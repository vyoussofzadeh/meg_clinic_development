Extracting Events from the STI Channel

 

Using MNE

1.   Load the raw file into mne_browse_raw using the following parameters on the command line:

--digtrig chan

--digtrgmask mask

where chan is STI101 or STI102 and mask is the mask you are using during data acquisition (example - 7)

 

2.   mne_browse_raw searches the STI channel for the events when loading the data.  A new file is then created with the extension –eve.fif

3.   You can also save all trigger line transitions to a text file for easier reading.

a.      File >> Save events (text)...

b.      Select one of the three options listed in the dialog box. 

c.      Give the file a name with the extension .eve

d.      Click OK

 

Events in the text MNE format (.eve):

 

Example:

21000 10.500       0   0

26653 13.326       0 1000

30588 15.294       0 1000

34906 17.453       0 1000

37898 18.949       0 1000

41261 20.631       0 1000

45046 22.523       0 1000

57025 28.513       0 1000

60687 30.344       0 1000

64034 32.017       0 1000

74798 37.399       0 2000

79430 39.715       0 2000

96697 48.348       0 2000

98869 49.435       0 2000

104622 52.311       0 2000

109555 54.778       0 2000

 

column 1 = sample time

column 2 = time (seconds)

column 3 = Old Bits (reference event)

column 4 = Event number

 

The first event in the file is an “offset” event.  This event represents the time during the acquisition between when “GO” was pressed and when “Record raw” was pressed.  All events in this file are offset by the first event.

See the MNE Users Manual for more information.

 

Using MEG-Clinic Tools

The tools for extracting events from a FIFF file are currently under development.  In the meantime a workflow is available for extracting events that are not compatible with the MNE format.

Create an Average using MaxFilter

1.   Open MaxFilter

2.   Load the *raw_sss.fif file

3.   Select Averager >> Load Raw Data...

4.   Select the *raw_sss.fif file again

5.   Click Execute

6.   When Complete, select File >> Show Log

7.   From the Log window, select Save

8.   In the current directory, name the file maxfilter_ave.log, click OK

9.   You can review this text file to verify that the correct events have been detected.

Extract Events from MaxFilter Log File

In Matlab:

The tools are contained in .../megclinic/tools and the MNE matlab toolbox.  Be sure these two folders are in your matlab path the follow the steps below.

 

1.   Define the average file (path and name):

>>aveFile = *_ave_sss.fif

 

2.   Parse the average file to get event descriptions

>>[Events, AcqPars] = parse_aveFile_events(aveFile);

 

3.   Define the number of average categories

>>AcqPars.nCategories = n;

 

4.   Define the raw file and the MaxFilter average log file (path and name)

>>rawFile = *raw_sss.fif;

>>logFile = .../maxfilter_ave.log;

 

5.   Read the average log file and create the event file

>>read_maxave_log

 

 

Averaging Events

Using MNE and MEG-Clinic

To average events contained in the raw file that has been encoded to be compatible with MNE

1.      Select a clean raw file from the database tree

2.      From the workflow buttons, select Average Clean Data

 

To average events in an MNE-formatted event file (.eve)

1.      Select a clean raw file from the database tree

2.      In the User Defined Events workflow panel, click Configure, a dialog will appear

3.      Select a protocol from the drop down box at the top. The field will be populated with the defaults for this protocol.  Protocols can be added to the list (see megclinic/config/task_defaults.xml)

4.      Select the event file (.eve) that contains the events

5.      Adjust the parameters as needed

6.      Click Average Events.  When averaging is complete the result files will be the average description file, .ave, the average log file, .log, and the averaged data file, _ave.fif.
